# 코트있음 UI 시각 감사

## 기존 화면 문제

- 알림 등록과 공지/사용방법이 예약 가능 결과보다 크게 노출되어 첫 화면의 목적이 흐렸음.
- 도시별 갱신 시각을 위해 메인 로드 후 Supabase를 4회 추가 조회했음.
- 용인시는 예약 가능 시간이 없어도 시설 카드가 노출되어 실제 예약 가능한 코트를 찾기 어려웠음.
- 긴 업데이트 이력과 기술 설명이 일반 사용자 화면에서 과하게 노출되었음.

## 변경 방향

- 메인 화면 우선순위를 `지역 선택 → 날짜 선택 → 시간 필터 → 결과 요약 → 예약 가능 코트`로 정리.
- 알림 설정은 접힘 영역으로 이동.
- 결과 위에 선택 조건, 예약 가능 시간 수, 시설 수, 가장 빠른 시간을 요약.
- 모바일에서 오늘/내일/모레 빠른 날짜 버튼을 제공.
- SEO와 AdSense 검토에 필요한 안내 콘텐츠는 하단 요약 블록과 기존 상세 페이지 링크로 보존.

## 모바일 문제와 대응

- 필터 입력을 유지하되 빠른 날짜 버튼을 추가해 날짜 선택 터치를 줄임.
- 알림 UI를 접어 첫 화면 높이를 줄임.
- 예약 가능 시간이 없는 시설 카드를 기본 결과에서 숨겨 스크롤 비용을 줄임.

## 광고 배치 검토

- 예약 버튼 내부, 필터 내부, sticky/floating 영역에는 광고를 두지 않는 전제를 유지.
- 광고가 추가될 경우 결과 요약과 첫 시설 카드 사이 또는 하단 콘텐츠 사이에 명확한 `aria-label="광고"` 섹션으로만 배치.
